# MindClaw Roadmap

## Phase 0 — Repository Foundation

- define project identity and public positioning
- create repository structure and initial documentation
- ship a minimal CLI entrypoint
- establish architecture boundaries

## Phase 1 — Core Domain Model

- define team, agent, task, and message models
- design runtime adapter interface
- define workspace and lifecycle abstractions
- add config and state layout

## Phase 2 — Local Orchestration MVP

- create teams from CLI
- spawn local workers
- assign tasks
- support status tracking and messaging
- add a simple terminal board view

## Phase 3 — Runtime Integration

- integrate a pluggable runtime adapter
- support session-aware workers
- support memory/tool/runtime capability reporting
- standardize worker bootstrap context

## Phase 4 — Developer Experience

- add examples and templates
- improve CLI ergonomics
- add architecture diagrams and tutorials
- improve tests and release workflow

## Phase 5 — Public Open Source Release

- polish README and screenshots
- publish first tagged release
- add contribution guide
- prepare demo scenarios for GitHub and social sharing
